# Миграция генератора отчётов со старой на новую архитектуру

## ✅ Что было выполнено

Успешно перенесен полнофункциональный генератор отчётов дисциплины со старой архитектуры (старая папка architectur/backup_main) на новую архитектуру с использованием декларативного фреймворка на Flask/SQLite.

### 1. **Данные и модели** (`pages/`)

Созданы новые Entity-модели для новой архитектуры:

- **`pages/completions/model.py`** - Completion: дневные записи (дата, номер дня, состояние, мысли, индекс трения)
- **`pages/completions/completion_habits.py`** - CompletionHabits: выполненные привычки на каждый день с их статистикой
- **`pages/habits/model.py`** - Habit: справочник привычек с категориями и характеристиками
- **`pages/combinations/model.py`** - Combination: сочетания привычек для бонусов

Все модели автоматически регистрируются в системе и создают таблицы SQLite при запуске.

### 2. **API endpoints** (`core/stats_api.py`)

Реализованы REST API для статистики:

- **GET /api/stats/period** - Статистика за период (неделя/месяц/всё время)
- **GET /api/stats/streaks** - Текущие стрики привычек  
- **GET /api/stats/daily_comparison** - Сравнение с предыдущим днём

Плюс автоматически сгенерированные CRUD API для каждой сущности:
- `/api/completions/{list,item,create,update,delete}`
- `/api/completion_habits/{list,item,create,update,delete}`
- `/api/habits/{list,item,create,update,delete}`
- `/api/combinations/{list,item,create,update,delete}`

### 3. **Фронтенд** (`static/`)

Перенесены и адаптированы все файлы фронтенда:

- **`report.html`** - Переработана HTML-страница с новой структурой (более чистая, без вложенных include'ов)
- **`report.js`** - Полностью переписан JavaScript для работы с новой архитектурой:
  - Парсинг текстового формата привычек (+ / - Название — количество единица [I] [S] и т.д.)
  - Взаимодействие с новыми API endpoints
  - Сохранение/загрузка из БД
  - Вычисление статистики и стриков
  - Работа с localStorage
- **`style.css`** - Обновлены стили для современного интерфейса

### 4. **Точка входа** (`app.py`)

Переработан файл app.py:
- Регистрация всех Entity моделей перед созданием БД
- Регистрация API blueprintов для каждой сущности  
- Регистрация статистики API
- Роуты для `/` и `/report` для страницы генератора

## 🚀 Как использовать

### 1. **Запуск приложения**
```bash
cd project
python app.py
```
Откройте браузер на `http://localhost:5000`

### 2. **Формат ввода привычек**

```
Здоровье
———————————————
+ Упражнения — 30 мин I[0.01] S[0.02] W[0.03] E[0] C[0] H[0.05] ST[1] $[0]
+ Пить воду — 2л I[0] S[0] W[0.01] E[0] C[0] H[0.02] ST[1] $[0]
- Сладкое I[0] S[0] W[0] E[-0.1] C[0] H[-0.05] ST[0] $[0]

Продуктивность
———————————————
+ Программирование — 2ч I[0.1] S[0.05] W[0.05] E[0.05] C[0] H[0] ST[0.5] $[100]
```

### 3. **Основные функции**

- ✏️ **Редактирование**: Нажмите ✎ на привычке для редактирования
- 📊 **Статистика**: Просмотр стриков и суммы характеристик
- 💾 **Сохранение**: В БД или localStorage
- 📈 **Периоды**: Неделя/месяц/всё время
- 🔄 **Загрузка**: Из БД по датам

### 4. **API примеры**

```bash
# Получить все привычки
curl http://localhost:5000/api/habits/list

# Создать привычку
curl -X POST http://localhost:5000/api/habits/create \
  -H "Content-Type: application/json" \
  -d '{"name":"Бег","category":"Спорт","i":0.1,"s":0.05}'

# Получить статистику
curl http://localhost:5000/api/stats/period?period=week

# Сохранить день
curl -X POST http://localhost:5000/api/completions/create \
  -H "Content-Type: application/json" \
  -d '{"date":"2026-03-22","day_number":100,"state":"WORK","friction_index":1}'
```

## 📝 Различия от старой архитектуры

| Старая | Новая |
|--------|-------|
| `discipline_days` таблица | `completions` + `completion_habits` (нормализованно) |
| Ручная регистрация API | Автоматическое создание через `register_entity_blueprint()` |
| `app2.py` 1000+ строк | `app.py` 50 строк + модули в `core/` и `pages/` |
| Смешанная логика | Разделение: модели (Entity), CRUD (db.py), API (api.py, stats_api.py) |
| SQL запросы в app | Объектный слой через Entity.from_row() |
